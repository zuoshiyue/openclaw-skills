# 小红书全自动图文 SOP（飞书版）

## 触发场景

- "帮我生成小红书图文"
- "把这个微信文章改成小红书格式"
- "小红书内容生产"
- "自动化图文生成"

---

## 完整流程

### 1. 输入处理

| 输入类型 | 处理方式 |
|---------|---------|
| 微信文章链接 | 解析正文内容 |
| 关键词/选题 | 搜索相关内容 |

**微信文章解析:**
```bash
curl -sL "https://r.jina.ai/https://mp.weixin.qq.com/s/xxx"
```

### 2. 内容重组

输出结构：

```markdown
### 标题（3-5 个备选）
- 包含关键词 + 情绪词 + 数字

### 分页设计
- 封面页：核心观点 + 吸引眼球
- 内容页 1-N：每页一个核心点
- 结尾页：总结 + 互动引导

### 每页内容
- 核心表达（1-2 句话）
- 配图建议（画面描述）
- 封面文案（大字标题）

### 标签建议
- 5-10 个相关 hashtag
```

### 3. 图片处理

**方案 A：飞书云文档上传（推荐）**
- 使用飞书 API 上传图片
- 获取 feishu:// 链接或公开链接

**方案 B：本地存储 + 飞书附件**
- 图片保存在 `~/.openclaw/workspace/images/`
- 作为附件上传到飞书多维表格

### 4. 数据存储

写入飞书多维表格：

| 字段 | 内容 |
|------|------|
| 选题来源 | 原始链接/关键词 |
| 标题草稿 | 生成的标题 |
| 正文内容 | 分页内容 |
| 图片附件 | 飞书附件/云文档链接 |
| 标签建议 | hashtag 列表 |
| 内容状态 | 待发布/已发布 |

## 注意事项

1. **避免生成文字** — AI 生成图片文字易出错
2. **尺寸建议** — 小红书 1080x1440 (3:4)
3. **标签数量** — 5-10 个相关 hashtag
4. **发布频率** — 避免短时间内大量发布
