## ❌ 问题原因

小红书 MCP 发布时，`\n` 换行符可能未被正确解析，导致内容挤在一起。

## ✅ 优化方案

**错误示例：**
```

```

**正确示例：**
```
content:"第一行
第二行

```

## 🔧 代码实现

### Python 格式化函数

```python
def format_xiaohongshu_content(text):
    """
    格式化小红书内容：
    1. 将 \n 转换为实际换行
    2. 在 emoji 后添加换行
    3. 控制段落间距
    """
    # 替换 \n 为实际换行
    text = text.replace('\\n', '\n')
    
    # 在 emoji 后确保有换行
    emojis = ['🎯', '✨', '📌', '✅', '💰', '🔧', '1️⃣', '2️⃣', '3️⃣']
    for emoji in emojis:
        text = text.replace(emoji, f'\n{emoji}')
    
    # 移除多余空行
    while '\n\n\n' in text:
        text = text.replace('\n\n\n', '\n\n')
    
    return text.strip()
```

### JavaScript 格式化函数

```javascript
function formatXiaohongshuContent(text) {
    // 替换 \n 为实际换行
    text = text.replace(/\\n/g, '\n');
    
    // 在 emoji 后添加换行
    const emojis = ['🎯', '✨', '📌', '✅', '💰', '🔧', '1️⃣', '2️⃣', '3️⃣'];
    emojis.forEach(emoji => {
        text = text.replace(new RegExp(emoji, 'g'), `\n${emoji}`);
    });
    
    // 移除多余空行
    text = text.replace(/\n{3,}/g, '\n\n');
    
    return text.trim();
}
```

## 🎯 最佳实践

1. **标题** - 20 字以内，包含数字和情绪词
2. **正文** - 每段 2-4 行，段间空行
3. **列表** - 用 emoji (✅⭐🔥) 开头
4. **标签** - 5-10 个，放在最后
5. **总长度** - 300-800 字最佳
