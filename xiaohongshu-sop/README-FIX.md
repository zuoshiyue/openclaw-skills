# 小红书发布格式问题修复

## ❌ 问题描述

**发布时间：** 2026-03-02 19:26  
**问题：** 发布的内容中 `\n` 换行符未被正确识别，导致内容挤在一起。

---

## 🔍 原因分析

小红书 MCP 在接收内容时，**字符串中的 `\n` 可能被当作字面字符**而非换行符处理。

**错误示例：**
```bash
content:"第一行\n第二行\n第三行"
# 实际发布：第一行\n第二行\n第三行（挤在一起）
```

---

## ✅ 解决方案

### 方案 1：使用实际换行符（推荐）

在 shell 脚本中使用单引号包裹内容，直接写入换行：

```bash
CONTENT='🚀 从 0 到 1！

✨ 不用写代码！

📌 核心功能：
✅ 功能 1
✅ 功能 2'

mcporter call 'xiaohongshu.publish_content' \
  title:"标题" \
  content:"$CONTENT" \
  ...
```

### 方案 2：使用 Python 格式化脚本

使用 `optimize-publish.py` 自动处理换行：

```bash
python3 optimize-publish.py
```

### 方案 3：使用格式化函数

在调用发布前，先用 Python 处理内容：

```python
from optimize-publish import format_xiaohongshu_content

content = format_xiaohongshu_content(raw_content)
```

## 🎯 最佳实践

1. **使用实际换行符** - 避免使用 `\n` 转义
2. **段间空行** - 每段之间保留空行
3. **emoji 分隔** - 用 emoji 作为视觉锚点
4. **控制长度** - 每行 20-30 字最佳
5. **标签集中** - 放在最后，用空格分隔
