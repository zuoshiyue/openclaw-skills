# 飞书文档默认权限配置指南

## 当前状态

⚠️ **API 限制：** 飞书开放平台的文档共享权限 API 需要特定的企业权限或在飞书开放平台控制台配置。

---

## 配置步骤

### 步骤 1：登录飞书开放平台

1. 访问 https://open.feishu.cn
2. 登录企业管理员账号
3. 进入「应用开发」→「企业自建应用」
4. 选择应用：`cli_a90024a2aafa9cc5`

### 步骤 2：配置应用权限

1. 进入「权限管理」标签
2. 搜索并添加以下权限：
   - ✅ 云文档只读权限
   - ✅ 云文档编辑权限
   - ✅ 云文档分享权限
   - ✅ 云文档协作管理权限

3. 点击「申请权限」
4. 等待管理员审批（通常立即生效）

### 步骤 3：配置文档默认权限

1. 在应用配置页面，找到「云文档」设置
2. 启用「创建文档时默认可编辑」
3. 或者设置「组织内获得链接的人可编辑」

### 步骤 4：测试权限

创建测试文档后，尝试：
1. 打开文档链接（不登录）
2. 检查是否有编辑权限
3. 或者分享给其他用户测试

---

## 临时解决方案

在 API 权限配置完成前，使用以下方式：

### 方案 A：手动设置一次默认权限

1. 创建任意飞书文档
2. 点击右上角「分享」
3. 点击「高级设置」
4. 设置「获得链接的人」为「可编辑」
5. 这样后续创建的文档会继承此设置

### 方案 B：机器人发送编辑链接

每次创建文档后，机器人发送：
```
📝 文档已创建：[文档标题]
🔗 链接：https://feishu.cn/docx/DOC_TOKEN
✏️ 权限：请点击链接后点击右上角"分享"→设置"组织内可编辑"
```

### 方案 C：使用飞书云文档模板

1. 创建一个模板文档
2. 设置模板为"组织内可编辑"
3. 后续基于模板创建文档

## 自动化脚本

创建文档后自动设置权限的脚本（需要 API 权限）：

```bash
#!/bin/bash
# feishu-set-permission.sh
DOC_TOKEN=$1
TOKEN=$(get_tenant_token)

curl -X POST "https://open.feishu.cn/open-apis/docx/v1/documents/$DOC_TOKEN/share" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"link_share_entity":"anyone","link_share_permission":"edit"}'
```
